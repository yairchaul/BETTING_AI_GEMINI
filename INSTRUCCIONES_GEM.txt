## REGLAS DEL PROYECTO

### Estructura de carpetas:
- /motors - Motores de análisis (MLB, NBA, UFC, Fútbol)
- /scrapers - Extracción de datos (ESPN, Caliente, Balldontlie)
- /visualizers - Interfaz de usuario (Streamlit)
- /utils - Utilidades (fuzzy matching, mapeo equipos)
- /data - Bases de datos y JSON

### Archivos clave que debes conocer:
- main_vision_completo.py - Orquestador principal
- visual_mlb.py - Visualización MLB (candidatos HR, strikes)
- visual_nba_mejorado.py - Visualización NBA (radar de triples)
- visual_ufc_final.py - Visualización UFC (datos físicos)
- visual_futbol_triple.py - Visualización fútbol (rachas)
- motors/motor_mlb_pro.py - Motor MLB (predicciones)
- motors/motor_over_under.py - Cálculo Over/Under
- radar_triples_nba.py - Radar de triples NBA
- balldontlie_client.py - API de estadísticas NBA

### Problemas conocidos a resolver:
1. Radar de triples NBA no muestra datos
2. Candidatos a Home Run no aparecen en MLB
3. Pitchers TBD en MLB (necesita scraper Caliente)
4. Gemini no conectado correctamente
5. Resultados se borran al cambiar de partido

### Formato de respuesta:
1. Siempre muestra el código exacto a modificar
2. Indica la ruta completa del archivo
3. Explica por qué la solución funciona
4. Usa colores en el código si es posible

### Reglas de codificación:
- Usar UTF-8 sin BOM
- Indentación de 4 espacios
- Importaciones desde la raíz o desde subcarpetas según estándar
- Mantener try-except para resiliencia
